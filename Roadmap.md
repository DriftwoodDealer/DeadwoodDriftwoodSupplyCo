# DEADWOOD: Technical Development Roadmap
## Phase 0 through Phase 4 Deployment Strategy

***

## Phase 0: The Core Foundation (Tonight's Goal)
Focus on establishing the brand presence and the base repository structure for GitHub Pages deployment.

### 0.1 Brand Sentinel
* Deploying the high-resolution DEADWOOD logo to a clean obsidian-toned splash page.
* Implementing the smokey glassmorphism background to establish the visual language immediately.
* Configuring the custom domain and SSL via GitHub Pages.

### 0.2 Repository Initialization
* Finalizing the .gitignore to prevent local 3D scan bloat.
* Establishing the README.md and Pricing.md as the primary project documentation.
* Setting up the directory structure for /scans, /assets, and /components.

***

## Phase 1: The Digital Shop & CMS
Transitioning from a splash page to a functional e-commerce environment using a high-end boilerplate.

### 1.1 Infrastructure Setup
* Integrating a Next.js 14+ boilerplate with Tailwind CSS and Framer Motion pre-configured.
* Connecting Supabase for the relational database (Inventory, GPS Data, User Auth).
* Configuring Cloudinary for automated 4K macro-photography delivery and background video hosting.

### 1.2 The "Visual Provenance" Engine
* Building the dynamic product page template with "Smokey Obsidian" styling.
* Implementing the Geotagging display to show GPS coordinates for every Missouri River find.
* Developing the "Bio-Sanctity" badge system to showcase kiln-bake certification.

***

## Phase 2: The "See It In Your Tank" Module
This is the "Force Multiplier" phase where we move into the 3D space.

### 2.1 The Rendering Engine
* Integrating Three.js and React Three Fiber (R3F) for the enclosure sandbox.
* Building the "Tank Architect" module with frosted glass overlays for Exo Terra and Zoo Med footprints.
* Implementing the WebGPU shaders for real-time shadows and depth.

### 2.2 The Gaussian Splat Pipeline
* Connecting the Luma AI / Polycam SDK for photorealistic driftwood rendering.
* Developing the "Environment Toggle" to swap lighting between Jungle Mist and High-Desert Sun.
* Building the "Spec-Check" tool to validate wood dimensions against selected tank sizes.

***

## Phase 3: The Contractor Portal & B2B
Establishing the revenue-driving infrastructure for professional clients.

### 3.1 Wholesale Architecture
* Building the "Contractor Dashboard" with a 40% discount logic for verified shops.
* Implementing the bi-monthly truckload management interface.
* Designing the "Private Gallery" for pre-drop access to "Masterpiece" grade logs.

### 3.2 The Custom Hunt System
* Developing the anatomical matching engine for taxidermists to upload post sketches.
* Building the "Commissioned Harvest" request portal with deposit integration.
* Integrating the "White Glove" shipping calculator for 100lb+ river stumps.

***

## Phase 4: Scaling & Marketing Automation
Optimizing the experience and expanding the "River Scout" narrative.

### 4.1 The Field Notes Blog
* Launching the "Grit of the Hunt" video series with cinematic river-run backgrounds.
* Implementing the "Drop Alert" SMS/Email system for Relic-grade discoveries.
* Building the "Provenance QR" generator for physical copper tags.

### 4.2 Performance Optimization
* Auditing 3D asset delivery to ensure lightning-fast loads on mobile devices.
* Finalizing the "Macro-Luxe" gallery for high-fashion landscaping presentations.
